# Локализация «Вавилон» — архитектура и добавление языков

Источник (ключи словаря) — **русский**. Перевод любого языка — это словарь
`I18N.<код> = { 'русская строка': 'перевод' }`. Базовый английский живёт в
`js/i18n.js` (`I18N.en`); каждый **новый** язык — отдельный файл
`js/lang/<код>.js`, подключаемый после `i18n.js`.

## TL;DR — «переведи на немецкий» в один прогон

Команда вида **«переведи игру на немецкий»** выполняется так (агентом, за один проход):

1. `node tools/i18n-export.js` — обновить `docs/i18n.reference.tsv` (две колонки: `ru` → `en`). Это эталон для перевода.
2. Перевести **все** ключи на целевой язык (опираясь на русскую строку и английский эталон). Собрать файл `js/lang/de.js` вида `I18N.de = { 'русская строка': 'Übersetzung', ... }` — **в том же наборе ключей**, что и `I18N.en`.
3. Зарегистрировать язык:
   - убедиться, что код есть в `LANG_META` (`js/i18n.js`); немецкий/испанский/французский/… уже добавлены;
   - добавить `<script src="js/lang/de.js?v=NN"></script>` в `index.html` сразу после строки-маркера `<!-- LANG-PACKS ... -->`.
4. `node check-i18n.js de` — должно быть «не хватает 0».
5. Поднять `?v=` у всех `<script>`/`<link>` в `index.html`, задеплоить.

Готово. Переключатель 🌐 и стартовый выбор языка **подхватят язык автоматически**
(список строится из загруженных словарей).

> Полуавтоматический путь: положить перевод в JSON `{ "ru": "перевод" }` и
> выполнить `node tools/i18n-add-lang.js de path/to/de.json` — он соберёт
> `js/lang/de.js`, проставит непереведённые ключи английским и отчитается о пропусках.

## Архитектура

```
js/i18n.js          ← движок (t/tp/L, localizeDOM, реестр LANG_META, en-словарь)
js/lang/de.js       ← I18N.de = { ... }   (подключается в index.html после i18n.js)
js/lang/es.js       ← I18N.es = { ... }
docs/i18n.reference.tsv   ← ru → en, эталон для переводчика/агента
docs/translation-keys.md  ← полный список ключей по разделам
docs/i18n.skeleton.json   ← пустой скелет { "ru": "" }
tools/i18n-export.js      ← пересобрать справочники из i18n.js
tools/i18n-add-lang.js    ← собрать js/lang/<code>.js из JSON-перевода
check-i18n.js             ← проверка полноты словаря языка
scan-keys.js              ← проверка, что все ключи в коде есть в словаре
```

### Как текст попадает к пользователю
| Механизм | Назначение |
|---|---|
| `t(s)` | перевод строки-ключа (`I18N[LANG][s]` или оригинал) |
| `tp(шаблон, vars)` | перевод шаблона + подстановка `{плейсхолдеров}` |
| `pushLog/clog(шаблон, vars)` | то же, сразу в журнал/боевой лог |
| `localizeDOM(root)` | после рендера переводит статичные текст-узлы, `placeholder`, `title`; умеет снимать **ведущий эмодзи** («🧝 Покои героя» → ключ «Покои героя») |
| `localizeClanEvent`, `_locBossName` | клиентский перевод **серверных** строк клана и имён рейд-боссов по шаблонам |

Все они работают через `t()` → **любой добавленный словарь применяется везде сразу**,
включая бой, логи, тосты, async-вставки (Арена/рынок/кланы локализуются после inject).

### Реестр языков (`js/i18n.js`)
```js
const LANG_META = { ru:{flag:'🇷🇺',name:'Русский'}, en:{flag:'🇬🇧',name:'English'},
  de:{flag:'🇩🇪',name:'Deutsch'}, es:{flag:'🇪🇸',name:'Español'}, ... };
```
`availableLangs()` = `ru` (исходник) + те коды, для кого **загружен** `I18N[code]`.
Поэтому пока нет `js/lang/de.js`, немецкого в пикере не будет — добавление чисто аддитивно.

## Правила перевода
1. **Плейсхолдеры `{...}` сохраняются дословно** (`{name}`, `{dmg}`, `{zone}`…); порядок можно менять, имена — нет.
2. **Эмодзи, числа, символы `«» × → −`, теги `<b>…</b>` сохраняются.**
3. **Не переводить:** имена игроков, названия кланов, пользовательские названия сборок/питомцев, текст «сообщения дня». Имена рейд-боссов переводятся (через словарь + `_locBossName`).
4. Апостроф в значении → пиши строку в двойных кавычках JS: `"...": "Hero's Chambers"`.
5. Грамматика простая (без склонений) — формулируй нейтрально; следи за длиной (узкий UI).

## Проверка
- `node check-i18n.js <code>` — полнота словаря языка против эталона `en` (показывает непереведённые/лишние).
- `node scan-keys.js <code>` — ищет ключи в коде (`t/tp/L/pushLog/clog/...`), которых нет в словаре языка.
- `node tools/i18n-export.js` — пересобрать `docs/*` после изменения ключей.

## Добавление НОВЫХ строк при разработке
1. Статичный текст пиши по-русски в HTML — `localizeDOM` переведёт по словарю.
2. С подстановкой → `tp('Куплено {n}× {res}', {n, res: rName(key)})` или `pushLog(...)`.
3. Имя из данных при отображении оборачивай `t(...)` (ключи/арт-файлы — не оборачивать).
4. Добавь новый русский ключ во **все** словари (`en` + `js/lang/*`); прогони `scan-keys.js`.
