# Фаза 5 — Пошаговый build-план: NFT-комплекты на TON + рынок на GetGems

> Это инженерный план «как и где сделать», без кода. Решения по продукту — в
> `docs/PHASE5_NFT.md` (§8): токенизируем **топ-снаряжение (мифик/комплекты)**,
> вторичка — **GetGems + роялти**, **юрпроверки пока нет → продажу/минт не
> запускаем**. Документ описывает, что нужно построить, когда гейт снимут.
> Все игровые куски при внедрении проходят правило «фича → энциклопедия → перевод».

---

## 1. Что именно токенизируем

**Единица NFT = целый комплект (сет) определённой рарности** — «комплектная вещь».
Один NFT представляет весь набор (5 частей одного класса), а не 5 отдельных токенов.

Почему комплект одним NFT (рекомендуется):
- чище экономика и рынок (торгуется как один лот, один поток роялти);
- меньше минтов/газа;
- совпадает с задачей «минт комплектных вещей».

Альтернатива (на будущее): каждая часть — отдельный NFT в коллекции. Гибче, но
дороже и сложнее в учёте сет-бонусов. **Старт — комплект одним NFT.**

В игре NFT-комплект даёт надетый набор (с **ограничением силы / «скин поверх»**,
чтобы не ломать баланс — см. §9).

---

## 2. Где делаем: стек и сервисы

| Слой | Чем | Заметки |
|---|---|---|
| Блокчейн | **TON** (mainnet) | единственный разрешённый для мини-аппов |
| Стандарты | **TEP-62** (NFT), **TEP-64** (метаданные), **TEP-66** (роялти) | роялти зашиты в стандарт |
| Контракты/минт | **@ton-community/assets-sdk** (npm) | `createNftCollection`, `mintNft`, работает с TON Connect; есть CLI |
| Контракты коллекции/продажи | **getgems-io/nft-contracts** | их `nft-collection`, `nft-item`, `nft-fixprice-sale-v3r3` |
| Метаданные | **Pinata (IPFS)** или наш домен Cloudflare | off-chain JSON по TEP-64 (1 ГБ на Pinata бесплатно) |
| Кошелёк игрока | **TON Connect** (`@tonconnect/ui`) + `ton_proof` | подключение, подпись-доказательство |
| Чтение ончейна | **tonapi.io** (или toncenter) | проверка владения NFT по адресу+коллекции |
| Рынок | **GetGems** | комиссия 5% + газ; роялти TEP-66 автоматически |

Минт-кошелёк (наш): отдельный TON-кошелёк-«минтер» с приватным ключом в секретах
воркера (как `BOT_TOKEN`). Деплоит коллекцию и минтит NFT игрокам.

---

## 3. Метаданные комплекта (TEP-64, off-chain JSON)

На каждый вид комплекта+рарность — JSON по адресу (IPFS/наш домен), напр.:

```json
{
  "name": "Звёздный аркан — Мифический комплект",
  "description": "Комплект мага из мира «Вавилон». Сет-бонус: +18% сила заклинаний и +60 макс. маны.",
  "image": "ipfs://.../arcane_mythic.png",
  "attributes": [
    { "trait_type": "Класс", "value": "Маг" },
    { "trait_type": "Рарность", "value": "Мифический" },
    { "trait_type": "Сет", "value": "Звёздный аркан" },
    { "trait_type": "Частей", "value": 5 },
    { "trait_type": "Сет-бонус", "value": "+18% spell, +60 MP" }
  ]
}
```

Коллекция тоже имеет свой JSON (имя, лого, описание, **роялти-адрес/процент**).
Картинки берём из существующих артов сетов (`img/…`), плюс «обложка» комплекта.

---

## 4. Контракт коллекции + роялти

1. Готовим метаданные коллекции (имя «Babylon Sets», лого, роялти **5–10%**,
   роялти-адрес = твой кошелёк-получатель).
2. Деплой **одной NFT-коллекции** (TEP-62/64) через `assets-sdk`
   (`createNftCollection`) с роялти-параметрами (TEP-66). Разовая операция, ~ доли TON.
3. Внутри коллекции — типы предметов = комплекты (по `content`/метаданным).

Роялти задаётся на уровне коллекции → действует на **все** перепродажи на любом
маркете (GetGems и др.) автоматически, бессрочно — **рекуррентный доход владельца**.

---

## 5. Минт: 3 модели (выбираем A + B)

**A. Первичная продажа (рекоменд., «доступный минт»):**
- В игре кнопка «Купить NFT-комплект» (за Stars или TON).
- Игрок подключил кошелёк (TON Connect). После оплаты **наш минт-кошелёк минтит NFT
  прямо на адрес игрока** (газ ~0.05 TON платим мы, заложен в цену).
- Для игрока это «оплатил → через минуту NFT в кошельке» — максимально просто.

**B. Токенизация заработанного комплекта:**
- Игрок, собравший полный сет в игре, жмёт «Перевести в NFT» → in-game комплект
  списывается → минтим NFT на его кошелёк. Даёт «настоящее владение» добытым.
- Газ: платим мы или игрок через TonConnect.

**C. (опц.) Пред-минт партии + продажа на GetGems** — лимитированные дропы. Позже.

Идемпотентность/учёт — в серверном журнале (как `wallet_`/ledger в воркере).

---

## 6. Привязка кошелька (под-фаза 5.1, делается первой)

- Клиент: `@tonconnect/ui`, кнопка «Подключить кошелёк» в Профиле/Банке, файл
  `tonconnect-manifest.json` на нашем домене.
- `ton_proof`: кошелёк подписывает челлендж от воркера → воркер проверяет подпись и
  сохраняет `wallet_<uid>.tonAddress`.
- Показ: адрес, баланс TON, NFT нашей коллекции у игрока.
- **Без активов и контрактов** — безопасный фундамент, не требует юр-гейта.

---

## 7. Связка «ончейн ↔ игра»

- NFT-эффект (надетый комплект) активен в игре, **пока кошелёк держит NFT**.
- Воркер на загрузке (и периодически) сверяет: tonapi → NFT коллекции у
  `tonAddress`. Нет NFT (продал/перевёл) — эффект снимается при следующей сверке.
- Кэшируем результат сверки (минуты), чтобы не дёргать API на каждый рендер.

---

## 8. Покупка-продажа на GetGems

**Как работает листинг (фикс-цена):**
- Продажа = перевод NFT на **sale-контракт** `nft-fixprice-sale-v3r3` (деплоится при
  выставлении). В контракте: цена, адрес+сумма комиссии маркета, **адрес+сумма
  роялти**. Условие: цена > роялти + комиссия.
- Покупка: транзакция на sale-контракт, `buyAmount = цена + ~1 TON` (лишнее
  возвращается). Маркету 5%, **роялти — тебе**, остальное продавцу.

**Что строим поэтапно:**
1. **MVP (рекоменд.):** в игре — кнопки **«Продать на GetGems»** и «Смотреть
   коллекцию» (deep-link вида `getgems.io/collection/<addr>` и `?modalNft=<addr>`).
   Сам трейд — на GetGems (их UI + кошелёк игрока). Роялти капает автоматически.
   Минимум кода, рынок и ликвидность — сразу.
2. **Позже (опц.):** свой in-app листинг/покупка через TON Connect, используя
   контракты GetGems (`nft-fixprice-sale-v3r3`) — полный контроль, но больше работы
   и поддержки.

---

## 9. Анти-pay-to-win и анти-абуз

- NFT-комплект даёт **ограниченную силу** (кап) или работает как **косметический
  «скин» поверх** обычной экипировки — чтобы NFT не давал прямого боевого
  преимущества за деньги (снижает и регуляторные риски).
- Лимиты на минт/частоту, проверка владения серверно, журнал операций.
- Никакого маркетинга «заработай реальные деньги».

---

## 10. Издержки и доход

- Деплой коллекции: разово, доли TON.
- Минт: ~**0.02–0.1 TON** за NFT (мы платим в модели A, закладываем в цену).
- Метаданные: Pinata 1 ГБ бесплатно (далее ~$0.5/ГБ) либо наш Cloudflare.
- GetGems: **5%** с продажи + газ.
- **Доход владельца:** первичная продажа (Stars/TON) + **роялти 5–10% с каждой
  перепродажи** (TEP-66, автоматически, бессрочно).

## 11. Порядок работ (вехи)

| Веха | Что | Кто |
|---|---|---|
| **M0. Юр-готовность** | юрпроверка (налоги/AML/статус актива), Terms | **владелец** (гейт) |
| **M1. Кошелёк (5.1)** | TON Connect + ton_proof, показ адреса/баланса/NFT | я (код) + ты (деплой воркера) |
| **M2. Подготовка ончейна** | завести минт-кошелёк, Pinata-ключ, GetGems-аккаунт; задать роялти-адрес | **ты** (ключи/аккаунты) |
| **M3. Коллекция** | метаданные коллекции + деплой NFT-коллекции (assets-sdk) с роялти | я (скрипт) + ты (подпись/средства) |
| **M4. Метаданные комплектов** | JSON+картинки по видам сет×рарность на IPFS/домен | я |
| **M5. Первичный минт (A)** | кнопка покупки NFT-комплекта (Stars/TON) → минт на кошелёк игрока; журнал; сверка владения | я (код+воркер) + ты (деплой/секрет ключа) |
| **M6. Рынок** | кнопки «Продать/Смотреть на GetGems» (deep-link) | я |
| **M7 (опц.)** | токенизация заработанного сета (B); свой in-app трейд | по желанию |

Энциклопедия + перевод на все языки — на шагах M1, M5, M6 (там, где UI).

---

## 12. Что нужно от владельца (решения/доступы)

1. **Юрпроверка (M0)** — блокирующий гейт реального минта/продажи.
2. **Минт-кошелёк TON** (новый, для проекта) + его сид во-secret воркера.
3. **Роялти-кошелёк** (куда идёт доход с продаж/роялти) и **процент** (реком. 7%).
4. **Pinata** аккаунт/ключ (или решение хостить метаданные на нашем Cloudflare).
5. **GetGems** аккаунт (для верификации коллекции, обложек, бейджа).
6. **Первичная цена** NFT-комплекта и валюта (Stars / TON).
7. **Модель силы NFT:** кап-сила или косметический скин (реком. косметика/кап).

> Готов начать с **M1 (подключение кошелька)** — он безопасен и не требует юр-гейта.
> Остальное (M3–M6) — после юрпроверки и получения ключей/аккаунтов.
